# JobSearch 产品规划 — 技能差距分析 MVP

## 一、产品定义

### 产品定位

**"先看市场，再投简历"** — 数据驱动的求职技能差距分析工具

### 一句话描述

输入目标岗位，系统基于真实招聘数据告诉你"市场需要什么"，再对比你的技能告诉你"还缺什么"。

### 核心价值主张

| 维度 | 传统方式 | JobSearch |
|------|----------|-----------|
| 信息来源 | 网上经验帖、培训班推荐 | 真实招聘数据统计 |
| 分析方式 | 主观判断、跟风学习 | 数据驱动、量化匹配 |
| 输出结果 | "你应该学XX" | "你缺XX，优先级是XX，市场要求率XX%" |
| 决策依据 | 别人说的 | 数据证明的 |

## 二、目标用户

### 主要用户画像

**用户A：转行求职者**
- 痛点：不知道目标岗位真正需要什么技能
- 场景：想从测试转数据分析，但不知道学什么
- 期望：明确知道"还差什么"，不浪费时间学没用的

**用户B：应届毕业生**
- 痛点：简历写满了技术，但不知道哪些值钱
- 场景：学了Python、Java、C++，但不确定哪个更匹配目标岗位
- 期望：知道优先展示哪些技能，重点准备什么

**用户C：在职跳槽者**
- 痛点：想知道自己的技能在市场上值多少
- 场景：做了3年后端，想知道跳槽需要补什么
- 期望：量化自己的市场竞争力

### 用户优先级

| 优先级 | 用户群 | 原因 |
|--------|--------|------|
| P0 | 转行求职者 | 痛点最强，付费意愿最高 |
| P1 | 应届毕业生 | 用户量大，传播性强 |
| P2 | 在职跳槽者 | 需求较弱，但LTV高 |

## 四、功能规格

### 功能1：岗位选择

**输入**：用户输入或选择目标岗位

**交互方式**：
- 搜索框：支持模糊搜索（如"后端"能匹配"Python后端开发"）
- 热门推荐：展示常被分析的岗位（如Python后端、数据分析、前端开发）
- 历史记录：展示用户之前分析过的岗位

**数据来源**：
- 已分析过的岗位从 `job_skills` 表的 `DISTINCT job_name` 获取
- 未分析过的岗位触发实时分析

**边界情况**：
- 岗位名太模糊（如"开发"）→ 提示用户具体化
- 该岗位无数据 → 触发实时分析，展示loading状态
- 分析失败 → 提示重试或换关键词

---

### 功能2：市场技能展示

**输入**：目标岗位名称

**输出**：该岗位的热门技能列表

**展示内容**：

| 字段 | 说明 | 示例 |
|------|------|------|
| 排名 | 按出现频率排序 | 1 |
| 技能名称 | 从JD中提取的技能关键词 | Python |
| 市场要求率 | 该技能在多少比例的JD中出现 | 95% |
| 趋势 | 相比上月的变化（可选，V2） | ↑ 上升 |

**交互**：
- 技能列表可滚动，展示 Top 15-20
- 每个技能前有勾选框
- 提供"全选"/"全不选"快捷按钮
- 鼠标悬停显示技能说明（可选，V2）

---

### 功能3：用户技能输入

**输入**：用户勾选自己掌握的技能

**交互方式**：
- **主方式**：在市场技能列表上直接勾选
- **补充方式**：底部提供文本框，用户可手动添加列表中没有的技能

**验证**：
- 至少选择1个技能才能提交
- 手动添加的技能需要去重检查

**状态保存**：
- 用户的技能选择保存在 session_state 中
- 切换岗位时清空选择

---

### 功能4：差距分析计算

**算法**：

```
输入：
  - 市场技能列表：[(skill_name, market_rate), ...]
  - 用户技能列表：[skill_name, ...]

处理：
  - matched = 市场技能 ∩ 用户技能
  - gap = 市场技能 - 用户技能
  - match_score = len(matched) / len(市场技能Top10) * 100

输出：
  - matched_skills: [(skill, market_rate), ...]
  - gap_skills: [(skill, market_rate), ...] 按market_rate降序
  - match_score: 百分比
```

**权重说明**：
- MVP阶段按简单的技能匹配数量计算
- V2可以引入加权计算（排名靠前的技能权重更高）

---

### 功能5：差距分析结果展示

**展示内容**：

#### 5.1 匹配度仪表盘
- 大数字展示：45%
- 颜色编码：
  - 绿色（70%+）：竞争力强
  - 黄色（40-70%）：需要提升
  - 红色（<40%）：差距较大

#### 5.2 技能对比表

| 你已掌握 | 市场要求率 | 状态 |
|----------|------------|------|
| Python | 95% | ✅ 强势匹配 |
| MySQL | 82% | ✅ 强势匹配 |
| Git | 45% | ✅ 基础匹配 |

| 你缺少的 | 市场要求率 | 优先级 |
|----------|------------|--------|
| Redis | 71% | 🔴 高优先 |
| Docker | 65% | 🔴 高优先 |
| FastAPI | 58% | 🟡 中优先 |
| Linux | 42% | 🟢 低优先 |

#### 5.3 学习路径建议
- 基于差距技能的市场要求率，给出学习顺序
- 每个技能附带简短的学习建议（可选，V2）

#### 5.4 操作按钮
- "重新分析"：回到岗位选择
- "分享结果"：生成分享链接或图片（V2）
- "保存报告"：下载PDF（V2）

## 七、技术方案

### 方案选择

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| 在现有Streamlit Chat UI中扩展 | 开发快，复用现有代码 | 交互受限于聊天形式 | ❌ 不推荐 |
| 新建独立Streamlit页面 | 开发快，UI灵活 | 与Chat UI分离 | ✅ MVP推荐 |
| 新建前端（React/Vue） | 体验最好 | 开发成本高 | ❌ V2再考虑 |

### MVP 技术方案

**前端**：Streamlit 多页面应用
- 新建 `pages/1_技能差距分析.py`
- 保持现有 Chat UI 作为"智能问答"功能

**后端**：复用现有数据层
- 读取 `job_skills` 表获取市场技能
- 新增 `user_skills` 和 `gap_reports` 表
- 差距计算逻辑放在 `memory/long_term.py` 或新建 `core/gap_analysis.py`

**数据库**：
- 复用现有 MySQL
- 新增2张表（自动创建）

## 十、迭代路线图

## 十一、风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| 用户不知道自己会什么 | 勾选不准确 | 提供技能说明、示例 |
| 市场技能数据不准 | 分析结果偏差 | 增加数据源、定期更新 |
| 岗位覆盖不足 | 用户搜不到目标岗位 | 支持触发实时分析 |
| 用户匹配度太低流失 | 产品价值感弱 | 强调"差距是正常的"，引导学习 |

## 十二、竞品对比

| 竞品 | 功能 | JobSearch优势 |
|------|------|---------------|
| 招聘网站（Boss直聘） | 展示JD原文 | 提炼技能排名，量化分析 |

| 职业规划文章 | 通用建议 | 个性化差距分析 |
| 简历优化工具 | 包装现有能力 | 告诉你缺什么，而非怎么包装 |
